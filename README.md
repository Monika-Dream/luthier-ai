# Luthier AI - 智能小提琴诊断实验室

## 项目简介

Luthier AI 是一个专为小提琴演奏者设计的智能诊断工具。它能够帮助演奏者识别和解决演奏过程中遇到的技术问题，包括身体姿势、手部位置、音质问题等。通过简单的交互式界面，用户可以快速获得专业的技术诊断和练习建议。

## 功能特性

### 1. 身体部位选择
- **左手问题**：小指、拇指、手腕、手指伸展等问题
- **右臂问题**：肘部、肩膀、前臂、弓控制等问题
- **姿势与弓法**：下巴托持、弓角度、背部紧张等问题
- **音质问题**：刺耳音、音弱、压力不均等问题

### 2. 智能诊断系统
- 根据用户选择的身体部位和具体问题点，生成个性化的诊断报告
- 提供根本原因分析
- 给出详细的解决步骤
- 包含实用的练习建议

### 3. 双模式设计
- **专注模式（Focus Mode）**：深色主题，适合集中注意力
- **放松模式（Relax Mode）**：柔和主题，适合轻松学习

### 4. 诊断历史记录
- 自动保存最近的诊断记录
- 可以快速查看和恢复之前的诊断结果

## 技术栈

- **框架**：Vue 3 (Composition API)
- **构建工具**：Vite
- **样式**：Tailwind CSS 4.x
- **图标**：Lucide Vue Next
- **路由**：Vue Router (已安装，可扩展)

## 使用方法

### 基本使用流程

1. **选择主要问题区域**
   - 在左侧面板中，点击四个主要类别之一：
     - 左手（Left Hand）
     - 右臂（Right Arm）
     - 姿势与弓法（Posture & Bow）
     - 音质（Sound Quality）

2. **选择具体问题点**
   - 选择主要区域后，会显示该区域下的具体问题点
   - 点击选择你遇到的具体问题

3. **描述问题**
   - 在文本框中详细描述你遇到的感觉或困难
   - 例如："小指无法够到G弦，需要过度伸展"

4. **生成诊断**
   - 点击"Generate Diagnosis"按钮
   - 系统会分析你的问题并生成诊断报告

5. **查看诊断结果**
   - 诊断报告包含：
     - 问题总结
     - 根本原因分析
     - 详细的解决步骤（5步）
     - 练习建议
     - 可视化参考说明

### 模式切换

- 点击右上角的模式切换按钮，可以在"专注模式"和"放松模式"之间切换
- 不同模式会改变背景颜色和整体视觉风格

### 查看历史记录

- 在左侧面板底部，可以看到最近3次的诊断记录
- 点击任意历史记录可以快速恢复查看

## 诊断数据库

当前系统内置了以下常见问题的诊断方案：

### 左手问题
- **小指问题**：肘部旋后问题，导致手型框架过小
- **拇指问题**：拇指紧张或位置不当
- **手腕问题**：手腕僵硬，影响换把和揉弦

### 右臂问题
- **肘部问题**：弓臂紧张和协调性问题
- **肩膀问题**：肩膀紧张，影响音质

### 音质问题
- **刺耳音**：弓压和角度问题
- **音弱**：弓压不足

### 默认诊断
如果选择的问题不在数据库中，系统会提供通用的技术改进建议，包括：
- 肌肉记忆训练方法
- 渐进式练习步骤
- 自我评估技巧

## 代码说明

### 主要状态管理

```javascript
// 在 App.vue 中定义的主要状态
const mood = ref('relax')              // 当前模式：'focus' 或 'relax'
const selectedArea = ref(null)          // 选中的主要区域ID
const selectedSubArea = ref(null)       // 选中的具体问题点ID
const issueDescription = ref('')        // 用户描述的问题
const showDiagnosis = ref(false)        // 是否显示诊断结果
const isLoading = ref(false)            // 是否正在加载
const diagnosis = ref(null)             // 当前诊断结果对象
const diagnosisHistory = ref([])        // 诊断历史记录数组
```

### 主要函数说明

#### `selectArea(areaId)`
- **功能**：选择主要问题区域
- **参数**：`areaId` (string) - 区域ID，如 'left-hand', 'right-arm' 等
- **返回值**：无
- **副作用**：会清空已选择的子区域

#### `selectSubArea(subAreaId)`
- **功能**：选择具体问题点
- **参数**：`subAreaId` (string) - 子区域ID
- **返回值**：无

#### `generateDiagnosis()`
- **功能**：生成诊断报告
- **参数**：无（使用当前选中的区域、子区域和问题描述）
- **返回值**：Promise（异步函数）
- **处理流程**：
  1. 设置加载状态为 true
  2. 模拟API调用（延迟1.8秒）
  3. 根据选择的区域和子区域组合键查找诊断数据
  4. 如果找不到匹配的诊断，使用默认诊断模板
  5. 保存诊断结果到历史记录
  6. 显示诊断结果

#### `resetForm()`
- **功能**：重置表单，开始新的诊断
- **参数**：无
- **返回值**：无
- **副作用**：清空所有选择状态和诊断结果

#### `restoreDiagnosis(item)`
- **功能**：恢复并显示历史诊断记录
- **参数**：`item` (object) - 诊断对象
- **返回值**：无

#### `toggleMood()`
- **功能**：切换专注模式和放松模式
- **参数**：无
- **返回值**：无

### 诊断数据格式

每个诊断对象包含以下字段：

```javascript
{
  title: string,           // 诊断标题
  issue: string,          // 问题描述
  rootCause: string,      // 根本原因分析
  steps: string[],        // 解决步骤数组（5个步骤）
  tip: string,           // 练习建议
  icon: Component,        // 图标组件
  visualLabel: string     // 可视化参考说明
}
```

## 样式系统

项目使用 Tailwind CSS 4.x，主要设计特点：

- **玻璃态效果**：使用 `backdrop-blur` 实现毛玻璃效果
- **渐变背景**：根据模式切换不同的渐变背景
- **过渡动画**：使用 Vue Transition 实现平滑的页面切换效果
- **响应式设计**：使用 Tailwind 的响应式类实现移动端适配

### 主要颜色主题

- **琥珀色（Amber）**：用于主要操作按钮和选中状态
- **玫瑰木色（Rosewood）**：用于渐变和强调
- **石板色（Slate）**：用于背景和文本
- **翠绿色（Emerald）**：用于成功状态和步骤指示
- **蓝色（Blue）**：用于提示信息

## 已知问题与改进方向

### 当前限制

1. **诊断数据有限**：目前只覆盖了部分常见问题，可以继续扩展诊断数据库
2. **模拟延迟**：当前使用固定的1.8秒延迟模拟API调用，实际应用中应连接真实API
3. **历史记录限制**：只保存最近3条记录，可以增加更多存储功能

### 代码优化建议

1. **组件拆分**：将 App.vue 拆分为更小的组件，提高可维护性
2. **状态管理**：考虑使用 Pinia 进行更复杂的状态管理
3. **类型安全**：添加 TypeScript 支持，提高代码质量
4. **测试覆盖**：添加单元测试和集成测试
